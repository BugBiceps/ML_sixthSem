Unit 1 Notes

Introduction to ML, Foundations and Data Preprocessing

# UNIT 1 — Introduction to Machine Learning ### Complete Detailed Notes with Diagrams, Tables & Flowcharts --- ## 1. Introduction to Machine Learning **Definition:** Machine Learning (ML) is a subfield of Artificial Intelligence (AI) that enables a system to automatically **learn patterns from data and improve performance on a task through experience (E)**, without being explicitly programmed for every rule. > **Tom Mitchell's Formal Definition (1997):** > "A computer program is said to **learn** from experience **E** with respect to some class of tasks **T** and performance measure **P**, if its performance at tasks in T, as measured by P, improves with experience E." **Why Machine Learning instead of Traditional Programming?** In traditional programming, a human writes explicit rules (logic) that convert Input → Output. In Machine Learning, the machine is given Input + Output (examples) and it **learns the rules/logic itself**. *(Diagram: In traditional programming Data + Rules → Output. In ML, Data + Output are fed in, and the machine derives the Rules/Model itself.)* ### Human Learning vs Machine Learning Analogy | Aspect | Human Learning | Machine Learning | |---|---|---| | Source of learning | Experience, teachers, books | Historical/labeled data | | Improves with | Practice | More training data / iterations | | Output | Skill / Knowledge | Trained Model (function) | | Errors corrected by | Feedback, correction | Loss function + optimization | ### Applications of Machine Learning - Email spam filtering, Recommendation systems (Netflix, Amazon) - Speech recognition (Siri, Alexa), Face recognition - Medical diagnosis, Fraud detection - Self-driving cars, Stock market prediction - Natural Language Processing (chatbots, translation) --- ## 2. Scope and Limitations of Machine Learning ### Scope (Advantages) 1. **Automation** of complex decision-making tasks. 2. Handles **large-scale, high-dimensional data** better than manual analysis. 3. **Continuous improvement** as more data becomes available. 4. Discovers **hidden patterns** not obvious to humans. 5. Wide applicability — healthcare, finance, agriculture, robotics, etc. ### Limitations 1. **Data dependency** — poor/insufficient/biased data → poor model. 2. **Black-box nature** — many models (deep neural nets) lack interpretability. 3. **Overfitting/Underfitting** — model may not generalize well. 4. **High computational cost** for training large models. 5. **Bias and fairness issues** — model may learn and amplify data biases. 6. Cannot handle tasks that require **true reasoning/common sense** the way humans do. 7. Requires **domain expertise** to correctly frame the problem and evaluate the model. --- ## 3. Regression **Regression** is a **supervised learning** technique used to predict a **continuous numerical output** based on one or more input features (predictors). ### 3.1 Simple Linear Regression Models the relationship between one independent variable (x) and a dependent variable (y) as a straight line: **y = β₀ + β₁x + ε** Where: - β₀ = intercept (value of y when x = 0) - β₁ = slope (change in y per unit change in x) - ε = error term (residual) **Cost Function (Mean Squared Error):** J(β₀, β₁) = (1/n) Σᵢ (yᵢ − ŷᵢ)² The best-fit line minimizes this cost using **Gradient Descent** or the **Normal Equation**. ### 3.2 Multiple Linear Regression Extends simple regression to multiple predictors: **y = β₀ + β₁x₁ + β₂x₂ + … + βₙxₙ + ε** ### 3.3 Logistic Regression (brief mention) Although named "regression," it is used for **classification** problems (binary output 0/1) using the **sigmoid function**: σ(z) = 1 / (1 + e^(−z)), where z = β₀ + β₁x ### Types of Regression — Summary Table | Type | Use Case | Output | |---|---|---| | Linear Regression | Predicting price, salary, temperature | Continuous | | Multiple Regression | Multiple predictors → single output | Continuous | | Polynomial Regression | Non-linear trend fitting | Continuous | | Logistic Regression | Spam/Not-spam, Pass/Fail | Categorical (0/1) | | Ridge/Lasso Regression | Regularized regression, avoids overfitting | Continuous | --- ## 4. Probability for Machine Learning Probability provides the mathematical foundation for reasoning about **uncertainty** in data and predictions. ### Key Concepts - **Random Variable:** A variable whose value is a numerical outcome of a random phenomenon. - **Probability Distribution:** Describes how probabilities are distributed over values of a random variable. - **Conditional Probability:** P(A|B) = P(A ∩ B) / P(B) — probability of A given B has occurred. - **Bayes' Theorem** (core to Naive Bayes classifiers): P(A|B) = [P(B|A) · P(A)] / P(B) - **Independence:** Two events A, B are independent if P(A ∩ B) = P(A)·P(B) - **Expectation (Mean):** E[X] = Σ x·P(x) - **Variance:** Var(X) = E[X²] − (E[X])² ### Why Probability Matters in ML | Use in ML | Example | |---|---| | Classification confidence | Predicting probability of "spam" = 0.87 | | Naive Bayes Classifier | Uses Bayes' theorem directly | | Generative models | GANs, VAEs model data distributions | | Uncertainty quantification | Confidence intervals on predictions | | Loss functions | Cross-entropy loss derived from likelihood | --- ## 5. Statistics for Machine Learning Statistics helps **summarize, interpret, and infer** conclusions from data. ### 5.1 Descriptive Statistics | Measure | Formula/Meaning | |---|---| | Mean (μ) | Σx / n (average value) | | Median | Middle value of sorted data | | Mode | Most frequently occurring value | | Variance (σ²) | Average squared deviation from mean | | Standard Deviation (σ) | √Variance — spread of data | | Range | Max − Min | | Skewness | Asymmetry of distribution | | Kurtosis | "Tailedness" of distribution | ### 5.2 Inferential Statistics Used to make predictions/generalizations about a **population** from a **sample**. - Confidence Intervals - Hypothesis Testing (covered in Section 10) - Correlation & Covariance (relationship between two variables) - p-value, significance level (α) ### Correlation Coefficient (Pearson's r) r = Σ[(xᵢ − x̄)(yᵢ − ȳ)] / √[Σ(xᵢ − x̄)² · Σ(yᵢ − ȳ)²] r ranges from −1 (perfect negative) to +1 (perfect positive correlation); r = 0 means no linear correlation. --- ## 6. Linear Algebra for Machine Learning Linear algebra provides the structural language for representing and manipulating data (vectors, matrices, tensors). ### Key Concepts | Concept | Meaning | ML Use | |---|---|---| | Scalar | A single number | Learning rate, loss value | | Vector | 1-D array of numbers | Feature vector (one data point) | | Matrix | 2-D array of numbers | Dataset (rows = samples, cols = features) | | Tensor | n-D generalization of matrix | Deep learning (images, batches) | | Dot Product | Σ aᵢbᵢ | Weighted sum in neurons | | Matrix Multiplication | Combines linear transformations | Forward pass in neural networks | | Transpose (Aᵀ) | Flips rows and columns | Used in normal equation, backprop | | Identity Matrix | Diagonal 1's | Neutral element in matrix multiplication | | Inverse Matrix (A⁻¹) | A·A⁻¹ = I | Solving linear regression (Normal Equation) | | Eigenvalues/Eigenvectors | Av = λv | PCA (dimensionality reduction) | | Norm (‖x‖) | Vector "length" | Regularization (L1, L2) | ### Normal Equation (Closed-form solution for Linear Regression) **β = (XᵀX)⁻¹ Xᵀy** --- ## 7. Convex Optimization Optimization is the process of **finding the parameters (weights) that minimize a cost/loss function**. Most ML training reduces to an optimization problem. **Convex Function:** A function where a line segment joining any two points on the curve lies **above or on** the curve. Convex functions have a **single global minimum** — no local minima to get trapped in. This is why many ML cost functions (e.g., MSE for linear regression) are designed/chosen to be convex. ### Gradient Descent Algorithm Iteratively updates parameters in the direction opposite to the gradient (steepest descent): **θ := θ − α · ∂J(θ)/∂θ** Where α = learning rate, J(θ) = cost function. ### Types of Gradient Descent | Type | Description | Pros/Cons | |---|---|---| | Batch Gradient Descent | Uses entire dataset per update | Stable but slow on big data | | Stochastic Gradient Descent (SGD) | Uses 1 sample per update | Fast, noisy convergence | | Mini-batch Gradient Descent | Uses small batch per update | Balance of speed & stability | ### Learning Rate Effects - **Too small α** → very slow convergence. - **Too large α** → may overshoot minimum / diverge. - **Well-tuned α** → smooth, efficient convergence. --- ## 8. Data Visualization Data visualization helps understand data distribution, relationships, and detect outliers **before** model building (Exploratory Data Analysis - EDA). ### Common Visualization Types | Chart | Purpose | |---|---| | Histogram | Shows frequency distribution of a numeric variable | | Box Plot | Shows median, quartiles, and outliers | | Scatter Plot | Shows relationship between two variables | | Bar Chart | Compares categorical data | | Line Chart | Shows trend over time | | Heatmap | Shows correlation matrix / intensity | | Pair Plot | Shows pairwise relationships across features | --- ## 9. Hypothesis Function and Testing ### 9.1 Hypothesis Function (in ML modeling context) The **hypothesis function h(x)** is the model's prediction function that maps inputs to outputs, parameterized by weights θ: **h_θ(x) = θ₀ + θ₁x₁ + θ₂x₂ + … + θₙxₙ** The learning algorithm's goal is to find θ values that make h_θ(x) as close as possible to the true output y, by minimizing the cost function J(θ). ### 9.2 Hypothesis Testing (Statistical Inference) Used to make a decision about a **population parameter** based on **sample data**. **Steps in Hypothesis Testing:** 1. State **Null Hypothesis (H₀)** — assumes no effect/difference. 2. State **Alternative Hypothesis (H₁ / Hₐ)** — what we want to prove. 3. Choose **significance level (α)** — typically 0.05. 4. Compute the **test statistic** (z-test, t-test, chi-square test, etc.) 5. Compute the **p-value**. 6. **Decision Rule:** If p-value < α → Reject H₀; else → Fail to reject H₀. ### Types of Errors in Hypothesis Testing | Error Type | Description | |---|---| | Type I Error (α) | Rejecting H₀ when it is actually true (false positive) | | Type II Error (β) | Failing to reject H₀ when it is actually false (false negative) | --- ## 10. Data Distributions A **probability distribution** describes how the values of a variable are spread/distributed. ### Common Distributions in ML | Distribution | Description | Example Use | |---|---|---| | Normal (Gaussian) | Symmetric bell curve, defined by mean μ & std dev σ | Heights, errors, most natural data | | Uniform | Every value equally likely | Random initialization of weights | | Binomial | Discrete, number of successes in n trials | Coin flips, pass/fail experiments | | Bernoulli | Special case of Binomial with n=1 | Binary classification outcome | | Poisson | Number of events in fixed interval | Number of website visits per hour | | Exponential | Time between events | Time until server failure | **Standard Normal Distribution:** A normal distribution with μ = 0 and σ = 1. Used to standardize data via **Z-score**: Z = (x − μ) / σ --- ## 11. Data Preprocessing Raw real-world data is often incomplete, noisy, and inconsistent. **Data preprocessing** converts raw data into a clean, usable format before feeding it to an ML model. ### 11.1 Key Steps **a) Handling Missing Data** - Remove rows/columns with missing values (if few). - Imputation: fill with mean/median/mode, or predictive imputation. **b) Handling Noisy Data** - Binning, regression smoothing, clustering to detect/remove outliers. **c) Data Integration** - Combining data from multiple sources into a consistent format. **d) Data Transformation** - Normalization, standardization, encoding categorical variables (One-Hot Encoding, Label Encoding). **e) Data Reduction** - Dimensionality reduction (PCA), feature selection, sampling. **f) Handling Categorical Data** | Technique | Description | |---|---| | Label Encoding | Assigns each category an integer (0,1,2…) | | One-Hot Encoding | Creates binary columns for each category | | Ordinal Encoding | Preserves rank order of categories | --- ## 12. Data Augmentation **Data Augmentation** artificially expands the size/diversity of a training dataset by creating modified copies of existing data — especially important when labeled data is limited (common in image/audio/text tasks). ### Common Techniques (by data type) | Data Type | Augmentation Techniques | |---|---| | Images | Rotation, flipping, cropping, zoom, brightness/contrast change, noise addition | | Text (NLP) | Synonym replacement, back-translation, random insertion/deletion | | Audio | Pitch shift, time stretch, adding background noise | | Tabular | SMOTE (Synthetic Minority Oversampling), noise injection | **Benefits:** Reduces overfitting, improves model generalization, balances imbalanced classes. --- ## 13. Normalizing Data Sets **Feature Scaling/Normalization** brings all features onto a similar scale so that no single feature dominates the model due to its magnitude (crucial for distance-based and gradient-based algorithms like KNN, SVM, Gradient Descent). ### 13.1 Min-Max Normalization Rescales data to a fixed range, usually [0, 1]: **x' = (x − x_min) / (x_max − x_min)** ### 13.2 Z-score Standardization Rescales data to have **mean = 0** and **standard deviation = 1**: **x' = (x − μ) / σ** ### 13.3 Comparison Table | Method | Formula | Range | When to Use | |---|---|---|---| | Min-Max Normalization | (x−min)/(max−min) | [0,1] | When data has known bounds, no major outliers | | Z-score Standardization | (x−μ)/σ | Unbounded (mean 0, std 1) | When data is normally distributed / has outliers | | Robust Scaling | (x−median)/IQR | Unbounded | Strong outliers present | --- ## 14. # UNIT 2 — Neural Networks & Deep Learning Fundamentals ### Complete Detailed Notes with Diagrams, Tables & Flowcharts --- ## 1. Linearity vs Non-Linearity **Linear Function:** A function where output changes proportionally with input — graphically a straight line. f(x) = wx + b. **Non-Linear Function:** Output does **not** change proportionally — graph is curved. Real-world data (images, speech, language) has complex, non-linear relationships that a purely linear model **cannot** capture. ### Why Non-Linearity is Essential in Neural Networks If every layer of a neural network only performed linear operations (wx + b), then **stacking multiple layers would be mathematically equivalent to a single linear layer** — no matter how many layers you add, the network could only learn straight-line (linear) decision boundaries. > **Key Insight:** Linear(Linear(x)) = Linear(x). Depth becomes meaningless without non-linearity. **Activation functions inject non-linearity**, allowing deep networks to approximate any complex function (Universal Approximation Theorem). --- ## 2. Activation Functions An **activation function** decides whether a neuron should "fire" and what signal to pass to the next layer, by transforming the weighted sum input **z = Σwᵢxᵢ + b** into an output. ### 2.1 Sigmoid Function **σ(z) = 1 / (1 + e⁻ᶻ)** — squashes input into range **(0, 1)**. - Used historically for binary classification output layer. - **Problem:** Vanishing gradient for large |z| (saturates), output not zero-centered, computationally expensive (exponential). ### 2.2 Tanh (Hyperbolic Tangent) **tanh(z) = (eᶻ − e⁻ᶻ) / (eᶻ + e⁻ᶻ)** — range **(−1, 1)**, zero-centered (better than sigmoid) but still suffers vanishing gradient at extremes. ### 2.3 ReLU (Rectified Linear Unit) **ReLU(z) = max(0, z)** - Most widely used activation in hidden layers of deep networks. - **Advantages:** Computationally cheap, faster convergence, reduces vanishing gradient problem for positive inputs. - **Problem: "Dying ReLU"** — neurons stuck outputting 0 for all inputs (if weights push z < 0 permanently, gradient = 0, neuron never updates again). ### 2.4 Leaky ReLU **f(z) = z if z > 0, else αz** (small α like 0.01) — allows a small non-zero gradient when z < 0, fixing the dying ReLU problem. ### 2.5 Softmax Used in the **output layer for multi-class classification** — converts a vector of raw scores (logits) into a probability distribution summing to 1: **Softmax(zᵢ) = e^zᵢ / Σⱼ e^zⱼ** ### Comparison Table | Activation | Range | Zero-Centered | Common Use | Key Issue | |---|---|---|---|---| | Sigmoid | (0,1) | No | Binary output layer | Vanishing gradient | | Tanh | (−1,1) | Yes | Hidden layers (older nets) | Vanishing gradient | | ReLU | [0,∞) | No | Hidden layers (most common) | Dying ReLU | | Leaky ReLU | (−∞,∞) | No | Hidden layers | Slight inconsistency | | Softmax | (0,1), sums to 1 | No | Multi-class output layer | Sensitive to large logits | --- ## 3. Weights and Bias A single artificial neuron computes: **z = w₁x₁ + w₂x₂ + … + wₙxₙ + b**, then applies activation: **a = f(z)** - **Weights (w):** Represent the **strength/importance** of each input feature's connection. Larger |w| means that input has more influence on the neuron's output. Learned during training. - **Bias (b):** An extra learnable parameter that shifts the activation function left/right, allowing the neuron to fit data that doesn't pass through the origin — gives the model additional flexibility. --- ## 4. Loss Function A **Loss Function** (or Cost Function) measures how far the model's predictions (ŷ) are from the actual target values (y). Training a neural network means **minimizing this loss**. ### Common Loss Functions | Loss Function | Formula | Used For | |---|---|---| | Mean Squared Error (MSE) | (1/n)Σ(y−ŷ)² | Regression | | Mean Absolute Error (MAE) | (1/n)Σ\|y−ŷ\| | Regression (robust to outliers) | | Binary Cross-Entropy | −[y·log(ŷ) + (1−y)·log(1−ŷ)] | Binary Classification | | Categorical Cross-Entropy | −Σ yᵢ·log(ŷᵢ) | Multi-class Classification | | Hinge Loss | max(0, 1 − y·ŷ) | SVM-based classifiers | **Loss vs Cost:** Loss = error for a **single** training example. Cost = **average loss** over the entire training/batch dataset: J(θ) = (1/m)Σ Loss(yᵢ, ŷᵢ) --- ## 5. Gradient Descent (in Neural Networks) Gradient Descent iteratively adjusts weights/bias to minimize the loss function by moving in the direction of **steepest decrease**: **w := w − α · ∂J/∂w**, **b := b − α · ∂J/∂b** Where α = **learning rate** (step size). ### Variants of Gradient Descent | Variant | Data used per update | Characteristic | |---|---|---| | Batch GD | Entire dataset | Stable, slow for large data | | Stochastic GD (SGD) | One sample | Fast but noisy | | Mini-batch GD | Small batch (e.g. 32, 64) | Best of both — standard in practice | --- ## 6. Multilayer Network (Multi-Layer Perceptron - MLP) A **Multilayer Network** consists of an **Input Layer**, one or more **Hidden Layers**, and an **Output Layer**, where each layer is fully (or partially) connected to the next via weighted edges, followed by activation functions. This layered, non-linear structure allows the network to learn hierarchical, complex representations of data — this is the foundation of **Deep Learning**. --- ## 7. Backpropagation **Backpropagation** is the algorithm used to **train** multilayer neural networks by efficiently computing the gradient of the loss function with respect to **every weight** in the network, using the **chain rule of calculus**, and propagating the error **backward** from the output layer to the input layer. ### Steps of Backpropagation 1. **Forward Pass:** Input passes through the network layer by layer to produce output ŷ. 2. **Compute Loss:** J = Loss(y, ŷ) 3. **Backward Pass:** Compute gradient of loss w.r.t. each weight, starting from the output layer, using the **chain rule**: ∂J/∂w = (∂J/∂a) · (∂a/∂z) · (∂z/∂w) 4. **Update Weights:** w := w − α·∂J/∂w (via gradient descent) 5. Repeat for many **epochs** until loss converges. --- ## 8. Weight Initialization Improper weight initialization can cause **slow convergence, vanishing/exploding gradients**. Weights should **not** be initialized to zero (all neurons would learn identical features) or with very large random values. ### Common Initialization Techniques | Method | Formula/Approach | Best Used With | |---|---|---| | Zero Initialization | All weights = 0 | ❌ Never used (symmetry problem) | | Random Initialization | Small random values (e.g., N(0, 0.01)) | Simple/shallow networks | | Xavier/Glorot Initialization | Variance = 1/n_in (or 2/(n_in+n_out)) | Sigmoid, Tanh activations | | He Initialization | Variance = 2/n_in | ReLU and its variants | **Xavier Initialization Formula:** W ~ U[−√(6/(nᵢₙ+nₒᵤₜ)), √(6/(nᵢₙ+nₒᵤₜ))] **He Initialization Formula:** W ~ N(0, 2/nᵢₙ) --- ## 9. Training and Testing | Phase | Purpose | Data Used | |---|---|---| | **Training** | Model learns weights/bias by minimizing loss via backpropagation & gradient descent | Training set | | **Validation** | Tune hyperparameters, detect overfitting during training | Validation set | | **Testing** | Final, unbiased evaluation of model performance on unseen data | Test set (never used in training) | ### Typical Data Split ### Overfitting vs Underfitting | Condition | Training Error | Test Error | Cause | Fix | |---|---|---|---|---| | Underfitting | High | High | Model too simple | Increase model complexity, train longer | | Good Fit | Low | Low | Balanced complexity | — | | Overfitting | Low | High | Model memorizes training data, poor generalization | Regularization, dropout, more data | --- ## 10. Unstable Gradient Problem (Vanishing / Exploding Gradients) As gradients are propagated backward through many layers (via the chain rule, which **multiplies** many partial derivatives together), they can become: - **Vanishing Gradients:** Gradients shrink exponentially as they propagate backward (especially with sigmoid/tanh, whose derivatives are always < 1). Early layers learn **extremely slowly or stop learning entirely**. - **Exploding Gradients:** Gradients grow exponentially large, causing **unstable updates**, weights become NaN/overflow, and training diverges. ### Solutions | Problem | Solutions | |---|---| | Vanishing Gradient | Use ReLU/Leaky ReLU, He/Xavier initialization, Batch Normalization, Residual/Skip connections (ResNets), LSTM/GRU (for RNNs) | | Exploding Gradient | Gradient Clipping, smaller learning rate, weight regularization, careful initialization | --- ## 11. Autoencoders An **Autoencoder** is an **unsupervised neural network** trained to reconstruct its own input, by first compressing it into a smaller latent representation (**Encoder**) and then reconstructing the original input from that representation (**Decoder**). ### Architecture - **Encoder:** Compresses input x into a lower-dimensional latent code z (bottleneck layer): z = f(x) - **Latent Space (Bottleneck):** Compressed representation — forces the network to learn only the most essential features. - **Decoder:** Reconstructs the output x̂ from z: x̂ = g(z) - **Loss (Reconstruction Error):** Measures difference between input x and output x̂, e.g., MSE = ‖x − x̂‖² ### Applications - Dimensionality reduction (non-linear alternative to PCA) - Denoising (Denoising Autoencoders remove noise from corrupted input) - Anomaly detection (high reconstruction error = anomaly) - Image compression, feature learning, pretraining for deep networks --- ## 12. Batch Normalization **Batch Normalization (BatchNorm)** normalizes the input to each layer (the activations) across a mini-batch, so they have **mean ≈ 0 and variance ≈ 1**, before scaling/shifting with learnable parameters. It is applied **between** the linear transformation and the activation function. **Formula:** x̂ᵢ = (xᵢ − μ_batch) / √(σ²_batch + ε), yᵢ = γ·x̂ᵢ + β Where γ (scale) and β (shift) are **learnable parameters**, and ε is a small constant to avoid division by zero. ### Benefits - Reduces **internal covariate shift** (the distribution of layer inputs changing during training). - Allows **higher learning rates** → faster training. - Acts as a **mild regularizer** (reduces need for Dropout in some cases). - Reduces sensitivity to weight initialization. --- ## 13. Dropout **Dropout** is a regularization technique where, during **each training iteration**, a random fraction (p) of neurons is **temporarily "dropped" (deactivated)** — their outputs set to zero — forcing the network to not rely too heavily on any single neuron and learn more **robust, redundant, generalized** features. - At **test time**, all neurons are active, but their outputs are scaled by (1−p) to compensate (or scaled during training — "inverted dropout"). - Typical dropout rate: **p = 0.2 to 0.5** --- ## 14. L1 and L2 Regularization **Regularization** adds a **penalty term** to the loss function to discourage overly complex models (large weights), reducing overfitting. **Total Cost = Original Loss + Regularization Penalty** ### L1 Regularization (Lasso) **J(θ) = Loss + λ·Σ|wᵢ|** - Adds penalty proportional to the **absolute value** of weights. - Tends to push some weights **exactly to zero** → performs automatic **feature selection**, produces sparse models. ### L2 Regularization (Ridge / Weight Decay) **J(θ) = Loss + λ·Σwᵢ²** - Adds penalty proportional to the **squared value** of weights. - Shrinks weights **towards zero but rarely to exactly zero** → keeps all features but reduces their influence, smoother solutions. Where **λ (lambda)** is the regularization strength — a hyperparameter controlling the trade-off between fitting the data and keeping weights small. ### L1 vs L2 Comparison | Aspect | L1 (Lasso) | L2 (Ridge) | |---|---|---| | Penalty | Σ\|w\| | Σw² | | Effect on weights | Some become exactly 0 | Shrunk but non-zero | | Feature selection | Yes (sparse model) | No | | Robust to outliers | More robust | Less robust | | Common name | Lasso Regression | Ridge Regression / Weight Decay | | Combined | **Elastic Net** = λ₁Σ\|w\| + λ₂Σw² (both penalties combined) | | --- ## 15. Momentum **Momentum** is an optimization technique that accelerates gradient descent by adding a fraction of the **previous update's direction** to the current update — like a ball rolling downhill, gaining speed and "remembering" past direction, which helps it move faster through flat regions and dampen oscillations in narrow ravines. **Formula:** vₜ = β·vₜ₋₁ + (1−β)·∂J/∂w (or vₜ = β·vₜ₋₁ + ∂J/∂w, depending on convention) w := w − α·vₜ Where **β** (momentum coefficient, typically 0.9) controls how much of the past gradient is retained. ### Related Advanced Optimizers | Optimizer | Key Idea | |---|---| | Momentum | Accumulates past gradients to accelerate convergence | | Nesterov Accelerated Gradient (NAG) | "Look-ahead" momentum — computes gradient at projected future position | | AdaGrad | Adapts learning rate per parameter (decreases over time) | | RMSProp | Adapts learning rate using moving average of squared gradients | | Adam | Combines Momentum + RMSProp — most popular in practice | --- ## 16. # UNIT 3 — Convolutional Neural Networks (CNN) ### Complete Detailed Notes with Diagrams, Tables & Flowcharts --- ## 1. Introduction to Convolutional Neural Networks (CNN) A **Convolutional Neural Network (CNN)** is a specialized deep learning architecture designed primarily for processing **grid-like data** such as images. Unlike a fully-connected MLP (which flattens an image into a 1-D vector and loses spatial structure), a CNN preserves the **spatial relationships** between pixels by using **convolution operations**, making it far more efficient and accurate for visual tasks. ### Why CNN over a normal Neural Network (MLP) for images? | Issue with MLP on Images | How CNN Solves It | |---|---| | Too many parameters (e.g., a 224×224×3 image = 150,528 inputs → huge fully-connected weight matrix) | **Parameter/Weight Sharing** — same small filter reused across the entire image | | Loses spatial structure (flattening destroys pixel neighborhood info) | Convolution operates on local regions, preserving spatial layout | | No translation invariance (object in a different position looks totally different to MLP) | Convolution + Pooling give **translation invariance** | ### Typical CNN Architecture (End-to-End Pipeline) --- ## 2. Convolution Layer The **Convolution Layer** is the core building block of a CNN. It applies a small matrix called a **filter/kernel** (e.g., 3×3 or 5×5) that **slides (convolves)** across the input image, computing the **element-wise dot product** at each position to produce a **feature map**. **Mathematical Operation:** (I * K)(i,j) = ΣₘΣₙ I(i+m, j+n) · K(m,n) Where I = input image, K = kernel/filter. ### Purpose - Each filter learns to detect a specific **feature** — edges, corners, textures in early layers; more complex shapes/objects in deeper layers. - Multiple filters in one layer produce multiple feature maps (stacked to form the output volume). ### Key Terms in a Convolution Layer | Term | Meaning | |---|---| | Filter / Kernel | Small learnable matrix (e.g. 3×3, 5×5) that slides over the input to detect features | | Feature Map / Activation Map | Output produced after convolving a filter over the input | | Receptive Field | Region of the input that influences a particular output neuron | | Depth (Number of filters) | Number of different filters used → determines number of output feature maps | | Weight Sharing | Same filter weights applied across the entire image → drastically fewer parameters than MLP | --- ## 3. Padding **Padding** adds extra pixels (usually zeros — "**zero-padding**") around the border of the input image **before** convolution. This is done because: 1. Without padding, the image **shrinks** after every convolution layer (since the filter cannot go beyond edges). 2. **Border/corner pixels** are used in fewer convolutions than central pixels, so information near edges is under-represented — padding fixes this. ### Types of Padding | Type | Description | Output Size Formula | |---|---|---| | **Valid Padding** (no padding) | No padding added; output shrinks | Output = (N − F)/S + 1 | | **Same Padding** | Padding added so output size = input size | Output = N (when S=1) | | **Full Padding** | Maximum padding so every pixel is convolved equally often | Output = N + F − 1 | --- ## 4. Stride **Stride (S)** defines the **number of pixels the filter moves** at each step while sliding across the input. - **Stride = 1:** Filter moves 1 pixel at a time → maximum overlap, larger output, more detailed feature map. - **Stride = 2 (or more):** Filter skips pixels → output is **downsampled** (smaller), less computation, less overlap. **Output Size Formula (general, combining padding P and stride S):** Output = ⌊(N − F + 2P) / S⌋ + 1 ### Effect of Stride & Padding on Output Size | Padding | Stride | Effect | |---|---|---| | Valid (0) | 1 | Output shrinks slightly each layer | | Same | 1 | Output size stays constant | | Valid/Same | 2+ | Output is downsampled (halved, etc.) — used instead of/along with pooling in some architectures | --- ## 5. Pooling Layer (Subsampling) **Pooling** (also called **Subsampling** or **Downsampling**) reduces the **spatial size (width × height)** of feature maps, which: 1. Reduces the number of parameters and computation in the network. 2. Controls **overfitting**. 3. Makes the network more **robust to small translations/distortions** in the input (translation invariance). ### Types of Pooling | Type | Operation | Effect | |---|---|---| | **Max Pooling** | Takes the **maximum** value in each window | Retains the most prominent/strongest feature (most common) | | **Average Pooling** | Takes the **average** value in each window | Smooths the feature map, retains general presence | | **Global Average Pooling (GAP)** | Averages the entire feature map into a single value per channel | Often replaces Flatten+Dense at the end of modern CNNs | ### Convolution vs Pooling — Quick Comparison | Aspect | Convolution Layer | Pooling Layer | |---|---|---| | Learnable parameters | Yes (filter weights, bias) | No (fixed operation) | | Purpose | Feature extraction | Dimensionality/spatial reduction | | Typical window | 3×3, 5×5 | 2×2 | | Output | Feature map | Downsampled feature map | --- ## 6. Flattening **Flattening** converts the final multi-dimensional feature map (e.g., a 7×7×64 tensor from the last conv/pool layer) into a **single, long 1-D vector**, so it can be fed into a fully-connected (Dense) layer for final classification/regression. **Example:** A feature map of shape (7, 7, 64) → flattened → a vector of length 7×7×64 = **3136**. --- ## 7. Loss Layer The **Loss Layer** (final layer of the network during training) computes the error between the network's predicted output and the true label, guiding backpropagation. | Task | Typical Loss Function used in Loss Layer | |---|---| | Binary Classification | Binary Cross-Entropy | | Multi-class Classification | Categorical Cross-Entropy (with Softmax output) | | Regression (e.g., bounding box coordinates) | MSE / Smooth L1 Loss | | Object Detection | Combination: classification loss + localization (bounding box) loss | The Loss Layer is not a "feature-extracting" layer like conv/pool — its sole role is to **quantify prediction error**, which is then backpropagated to update all preceding layers' weights. --- ## 8. Dense Layer (Fully Connected Layer) A **Dense Layer** (Fully Connected / FC layer) connects **every neuron** in it to **every neuron** in the previous layer — same as a standard MLP layer. In a CNN, Dense layers usually appear **after** the flattening step, at the **end** of the network, to combine the extracted features and produce the final classification/regression output. **Computation:** y = f(Wx + b), where every element of x connects to every neuron. ### Role in CNN Architecture - Convolution + Pooling layers → **extract features** (edges, textures, shapes, objects) - Flatten → converts to vector - Dense Layer(s) → **combine features and make the final decision** (e.g., softmax output layer for classification) | Layer Type | Parameters | Role | |---|---|---| | Convolution | Few (shared filter weights) | Local feature detection | | Pooling | None | Downsampling | | Dense (FC) | Many (every connection is a weight) | Global reasoning / final classification | --- ## 9. 1×1 Convolution A **1×1 Convolution** (also called **Network-in-Network** or pointwise convolution) uses a filter of size 1×1 instead of the usual 3×3/5×5. Although it doesn't look at any spatial neighborhood (only a single pixel position across all channels), it is extremely useful for: 1. **Dimensionality Reduction (channel reduction):** Reduces the **number of channels/depth** without changing spatial (H×W) dimensions — e.g., reducing 256 channels to 64 channels, drastically cutting computation for subsequent larger convolutions. 2. **Adding Non-linearity:** Each 1×1 conv is typically followed by an activation (ReLU), adding extra non-linear transformation without changing spatial size. 3. **Cross-channel information mixing:** Combines information across channels at the same spatial location. --- ## 10. Inception Network (GoogLeNet) The **Inception Network** addresses the problem of choosing the "right" filter size (1×1 vs 3×3 vs 5×5) at each layer by applying **multiple filter sizes in parallel** within the same layer (an "**Inception Module**"), then **concatenating** all the resulting feature maps together — letting the network learn which combination works best. ### Inception Module Structure Each Inception Module runs these branches **in parallel** on the same input, then concatenates outputs along the channel dimension: 1. 1×1 convolution 2. 1×1 convolution → 3×3 convolution 3. 1×1 convolution → 5×5 convolution 4. 3×3 max pooling → 1×1 convolution > The **1×1 convolutions before the 3×3/5×5** convolutions act as "**bottleneck layers**" — they reduce the number of channels first, dramatically cutting computational cost before the more expensive larger convolutions. **Key Insight:** By concatenating outputs from multiple filter sizes, the network becomes **wider** (more feature diversity per layer) instead of only deeper, improving accuracy while keeping computational cost manageable via 1×1 bottlenecks. --- ## 11. Input Channels **Channels** represent the **depth** dimension of an image/feature map — the number of separate "layers" of information at each spatial location. | Image Type | Number of Input Channels | Example | |---|---|---| | Grayscale image | 1 | Single intensity value per pixel | | RGB Color image | 3 | Red, Green, Blue values per pixel | | RGBA image | 4 | RGB + Alpha (transparency) | | Feature map (mid-network) | Any (e.g., 64, 128, 256…) | Number of filters used in the previous conv layer | **Convolution across channels:** A filter applied to a multi-channel input **must have the same depth as the input** (e.g., a 3×3 filter on an RGB image is actually 3×3×3), and it computes a **single combined value** by summing the convolution results across all input channels. --- ## 12. Transfer Learning **Transfer Learning** reuses a model that has **already been trained on a large dataset** (e.g., ImageNet with 1.4M images, 1000 classes) and adapts it to a **new, related task** — usually with much less data and training time, since the pretrained model already knows general low/mid-level visual features (edges, textures, shapes). ### Why it works Early CNN layers learn **generic features** (edges, colors, textures) that are useful across almost any visual task. # Machine Learning — Unit 4 Study Notes ## Sequence Models (RNN/LSTM/GRU), Attention, Machine Translation & Reinforcement Learning --- ## 1. Introduction to Sequence Models ### 1.1 Why Not Plain Feedforward Networks? Standard feedforward/CNN architectures assume **fixed-size, independent** inputs. But data like text, speech, and time-series is **sequential** — the order matters, inputs can be variable-length, and there are long-range dependencies between elements (e.g., a pronoun referring back to a noun many words earlier). **Recurrent Neural Networks (RNNs)** were designed specifically to process sequences by maintaining a **hidden state** that carries information from earlier time steps forward. ### 1.2 Notation Used Throughout This Unit ``` x = input at time step t (e.g., one word/embedding) y = output/prediction at time step t a = hidden activation (memory) at time step t Tx = length of input sequence Ty = length of output sequence Wax, Waa, Wya = weight matrices; ba, by = bias vectors ``` --- ## 2. Recurrent Neural Networks (RNN) ### 2.1 Core Idea At every time step, the RNN takes the **current input** and the **previous hidden state**, and produces a **new hidden state** (and optionally an output). The *same* weights are reused (shared) at every time step — this weight-sharing is what lets the network generalize across sequence positions and handle variable-length input.
### 2.2 Forward Propagation Equations ``` a = g1( Waa · a + Wax · x + ba ) -- typically g1 = tanh y = g2( Wya · a + by ) -- g2 = softmax/sigmoid ``` ### 2.3 Backpropagation Through Time (BPTT) Since the same weights are reused at every step, the loss gradient must be summed across **all** time steps and propagated backward through the entire unrolled chain: ``` L = Σ over t of L(y_hat, y) ∂L/∂W = Σ over t of ∂L/∂W (chain rule unrolled back through every a) ``` ### 2.4 The Vanishing/Exploding Gradient Problem Because the same weight matrix `Waa` is multiplied repeatedly across many time steps during backpropagation, gradients shrink toward zero (**vanishing**) or grow unboundedly (**exploding**) as sequence length increases. | Problem | Effect | Typical Fix | | :--- | :--- | :--- | | **Vanishing Gradient** | Network cannot learn **long-range dependencies** — early time steps stop receiving meaningful gradient signal. | LSTM / GRU gating; residual/skip connections. | | **Exploding Gradient** | Weight updates become huge, training diverges (loss → NaN). | **Gradient clipping** (rescale gradient if its norm exceeds a threshold). | ### 2.5 Types of RNN Architectures (by input/output length) | Architecture | Tx vs Ty | Example Use Case | | :--- | :--- | :--- | | **One-to-One** | Tx = Ty = 1 | Plain feedforward network (not really recurrent). | | **One-to-Many** | Tx=1, Ty>1 | Music/text generation from a seed. | | **Many-to-One** | Tx>1, Ty=1 | Sentiment classification of a sentence. | | **Many-to-Many (equal)** | Tx = Ty | Named-entity recognition (tag every word). | | **Many-to-Many (unequal)** | Tx ≠ Ty | Machine translation (encoder-decoder, Section 5). | --- ## 3. Long Short-Term Memory (LSTM) ### 3.1 Motivation LSTM was designed explicitly to solve the vanishing-gradient/long-range-dependency problem of plain RNNs by introducing a **cell state** (`c`) that acts as a conveyor belt of memory, modified only through carefully controlled **gates**, rather than being squashed through a `tanh`/matrix-multiply at every single step. ### 3.2 The Three Gates + Candidate | Gate | Formula | Purpose | | :--- | :--- | :--- | | **Forget Gate (Γf)** | `Γf = σ(Wf·[a, x] + bf)` | Decides what fraction of the old cell state to **discard**. | | **Update/Input Gate (Γu)** | `Γu = σ(Wu·[a, x] + bu)` | Decides how much of the new candidate value to **write in**. | | **Output Gate (Γo)** | `Γo = σ(Wo·[a, x] + bo)` | Decides how much of the cell state to **expose** as the hidden output. | | **Candidate Cell Value** | `c̃ = tanh(Wc·[a, x] + bc)` | Proposed new content to potentially add to memory. | ### 3.3 State Update Equations ``` c = Γf * c + Γu * c̃ -- forget old + write new (elementwise *) a = Γo * tanh(c) -- gated exposure of memory as hidden state ```
### 3.4 Why This Solves Vanishing Gradients The cell-state update `c = Γf * c + Γu * c̃` is **additive**, not a repeated matrix-multiply/squash. When `Γf ≈ 1`, gradient can flow backward through many time steps nearly unchanged (like a "gradient highway"), letting the network learn dependencies spanning hundreds of steps. --- ## 4. Gated Recurrent Unit (GRU) ### 4.1 Motivation GRU is a **simplified** alternative to LSTM (introduced by Cho et al., 2014) that merges the cell state and hidden state into a single vector and uses only **two gates** instead of three — fewer parameters, faster to train, and often comparable performance. ### 4.2 GRU Equations ``` Γr = σ(Wr·[a, x] + br) -- Reset gate: how much of past to forget when forming candidate Γu = σ(Wu·[a, x] + bu) -- Update gate: how much of past to keep vs. replace ã = tanh(W·[Γr * a, x] + b) -- Candidate activation a = Γu * a + (1 - Γu) * ã -- Interpolate between old state and candidate ``` ### 4.3 LSTM vs GRU — Comparison Table | Parameter | LSTM | GRU | | :--- | :--- | :--- | | **Gates** | 3 (forget, update, output) | 2 (reset, update) | | **States Maintained** | Separate cell state `c` and hidden state `a` | Single merged state `a` | | **Parameters** | More (larger, more expressive) | Fewer (~25% less, faster to train) | | **Typical Performance** | Slightly better on very long sequences / large datasets | Comparable, often preferred for smaller datasets or speed-critical tasks | | **Year Introduced** | Hochreiter & Schmidhuber, 1997 | Cho et al., 2014 | --- ## 5. Sequence-to-Sequence Models & Machine Translation ### 5.1 The Encoder-Decoder Architecture A **machine translation** system typically uses two RNNs (usually LSTMs/GRUs): an **Encoder** that reads the entire source sentence and compresses it into a fixed-length context vector, and a **Decoder** that generates the target sentence one word at a time, conditioned on that context.
### 5.2 Conditional Language Model View Machine translation can be viewed as a **conditional language model**: instead of modeling `P(y<1>,...,y)` from scratch, the decoder models `P(y<1>,...,y | x<1>,...,x)` — the probability of the target sentence *given* the source sentence (encoded as the context). ### 5.3 Why Greedy Decoding Fails Picking the single most-likely word at each decoder step (**greedy search**) does **not** produce the overall most-likely sentence, because an early locally-optimal word choice can lock the model into a globally suboptimal sentence. This motivates **Beam Search** (Section 6). --- ## 6. Beam Search and Beam Width ### 6.1 What is Beam Search? **Beam search** is an approximate search algorithm that, instead of keeping only the single best partial sentence (greedy) or exploring *all* possible sentences (exhaustive, exponential), keeps track of the **top B most likely partial sequences** at every decoding step, where **B is the beam width**. ### 6.2 Algorithm Steps ``` 1. At step 1: compute P(y<1> | x) for every word in vocabulary; keep the top B candidates. 2. At step t: for EACH of the B partial sequences kept so far, expand with every possible next word, compute cumulative probability P(y<1>..y | x), keep only the overall top B sequences (out of B × |vocab| candidates). 3. Repeat until each beam hits an token or max length. 4. Return the completed sequence with highest overall score. ```
### 6.3 Beam Width (B) — The Key Hyperparameter | B value | Behavior | Trade-off | | :--- | :--- | :--- | | **B = 1** | Reduces exactly to **greedy search**. | Fastest, but lowest translation quality. | | **Small B (e.g., 3–10)** | Good balance for production systems. | Reasonable quality, manageable compute/memory. | | **Large B (e.g., 1000+)** | Approaches exhaustive/optimal search. | Diminishing quality returns, high compute/memory cost. | ### 6.4 Length Normalization Raw joint probability `Π P(y | ...)` is a product of numbers < 1, so it **shrinks as sentence length grows**, unfairly penalizing longer (often correct) translations. Beam search therefore typically maximizes a **length-normalized log-probability objective**: ``` Score(y) = (1/Ty^α) · Σ over t of log P(y | x, y<1>...y) ``` where `α` (commonly ≈ 0.7) softens the normalization; using log-probabilities also avoids numerical underflow from multiplying many small probabilities together. ### 6.5 Error Analysis with Beam Search When a translation model produces a poor output, the fault could lie in the **RNN model** or the **beam search algorithm**. Compare `P(y* | x)` (human/reference translation) against `P(ŷ | x)` (model output): | Case | Diagnosis | Fix Direction | | :--- | :--- | :--- | | `P(y*) > P(ŷ)` | Beam search failed to find the higher-probability sequence the model *could* have produced. | Increase beam width. | | `P(y*) ≤ P(ŷ)` | The RNN model itself assigns wrong/lower probability to the correct translation. | Improve/retrain the model (architecture, data, regularization). | --- ## 7. BLEU Score ### 7.1 The Evaluation Problem Unlike classification, machine translation has **no single correct answer** — many different sentences can be equally good translations. **BLEU (Bilingual Evaluation Understudy)** score gives an automated way to numerically compare a machine-generated translation against one or more human reference translations. ### 7.2 Modified N-gram Precision Plain precision (fraction of generated words that appear in the reference) can be gamed — e.g., outputting the single most common word repeatedly. BLEU uses **modified/clipped precision**: for each n-gram in the candidate, its count is **clipped** to the maximum number of times that n-gram appears in *any single* reference. ``` p_n = ( Σ over n-grams in candidate of clipped_count(n-gram) ) ─────────────────────────────────────────────────────── ( Σ over n-grams in candidate of count(n-gram) ) ``` ### 7.3 Brevity Penalty (BP) Modified precision alone rewards very short outputs (a short candidate matching a few reference words scores high precision). BLEU introduces a **brevity penalty** to punish translations that are too short compared to the reference length: ``` BP = 1 if candidate_length (c) > reference_length (r) BP = exp(1 - r/c) if c ≤ r ``` ### 7.4 Full BLEU Formula ``` BLEU = BP × exp( Σ(n=1 to N) w_n · log(p_n) ) Typically: N = 4 (unigrams through 4-grams), w_n = 1/N (equal weights) ``` ### 7.5 Worked Example (Conceptual) ``` Reference: "the cat is on the mat" Candidate: "the cat the cat on the mat" Unigram "the": appears 3× in candidate, but only 2× in reference → clipped count for "the" = 2 (not 3) → this clipping is what BLEU's precision fixes. ``` ### 7.6 Strengths & Limitations of BLEU | Strengths | Limitations | | :--- | :--- | | Fast, automated, no human evaluators needed per run. | Ignores meaning/semantics — purely surface n-gram overlap. | | Correlates reasonably well with human judgment in aggregate. | Penalizes valid paraphrases/synonyms not present in reference(s). | | Works with multiple reference translations for flexibility. | Poor signal at the single-sentence level; more reliable in aggregate over a corpus. | --- ## 8. The Attention Model ### 8.1 The Bottleneck Problem The basic encoder-decoder (Section 5) compresses the **entire** source sentence into one fixed-length context vector — for long sentences this is an information bottleneck, and translation quality measurably degrades as sentence length grows. ### 8.2 Core Idea of Attention Instead of using a single fixed context vector for the whole output sentence, the **attention mechanism** lets the decoder, at every output time step, compute a **weighted combination of ALL encoder hidden states**, dynamically "attending" more strongly to the source words most relevant to the word currently being generated.
### 8.3 Attention Weight Computation ``` e = attention-score( s, a ) -- small NN (or dot-product) scoring -- how relevant source position t' is -- for producing output position t α = softmax over t' of e -- normalize scores into weights that sum to 1 context = Σ over t' of α · a -- weighted sum of ALL encoder states ``` ### 8.4 Interpretation The attention weights `α` form a soft alignment: `α` represents how much attention output word `t` should pay to input word `t'`. Visualizing this as a heat-map matrix produces an interpretable **alignment matrix**, often roughly diagonal for languages with similar word order. ### 8.5 Computational Cost Attention over a sequence of length `Tx` for every decoder step costs **O(Tx × Ty)** — quadratic in sequence length — considerably more expensive than the constant-size context vector of the vanilla encoder-decoder, but the quality gain (especially on long sequences) is usually well worth the cost. --- ## 9. Reinforcement Learning — Framework & Foundations ### 9.1 What is Reinforcement Learning? **Reinforcement Learning (RL)** is a paradigm where an **agent** learns to make a sequence of decisions by interacting with an **environment**, receiving **rewards** (or penalties) as feedback, and adjusting its behavior (**policy**) to maximize its **cumulative long-term reward** — unlike supervised learning, there is no labeled "correct action" given directly; the agent must discover good behavior through trial and error. ### 9.2 The Agent-Environment Loop
### 9.3 Key RL Terminology | Term | Meaning | | :--- | :--- | | **State (s)** | A description of the current situation the agent observes. | | **Action (a)** | A choice the agent can make from the available action space. | | **Reward (r)** | Immediate numeric feedback signal received after taking an action. | | **Policy (π)** | The agent's strategy — a mapping from states to actions (or a probability distribution over actions), `π(a\|s)`. | | **Value Function (V)** | Expected cumulative future reward starting from a state, following a given policy. | | **Q-Function (Q)** | Expected cumulative future reward starting from a state, taking a specific action, then following a policy. | | **Episode** | One complete run from an initial state to a terminal state. | | **Discount Factor (γ)** | A value in `[0,1]` that down-weights future rewards relative to immediate ones. | ### 9.4 Exploration vs. Exploitation A fundamental RL trade-off: **exploitation** means choosing the action currently believed to be best (to maximize immediate expected reward), while **exploration** means trying lesser-known actions to discover potentially better long-term strategies. A common simple strategy is **ε-greedy**: choose the best-known action with probability `1-ε`, and a random action with probability `ε`. --- ## 10. Markov Decision Process (MDP) ### 10.1 Definition An **MDP** formalizes the RL environment as a tuple **(S, A, P, R, γ)**: | Component | Meaning | | :--- | :--- | | **S** | Set of all possible states. | | **A** | Set of all possible actions. | | **P(s'\|s,a)** | Transition probability — chance of landing in state `s'` after taking action `a` in state `s`. | | **R(s,a,s')** | Reward function — immediate reward received for that transition. | | **γ** | Discount factor controlling how much future reward matters relative to immediate reward. | ### 10.2 The Markov Property The defining assumption of an MDP: **the future depends only on the current state and action, not on the full history of how the agent arrived there** — i.e., `P(s_{t+1} | s_t, a_t, s_{t-1}, a_{t-1}, ...) = P(s_{t+1} | s_t, a_t)`. This memoryless property is what makes MDPs mathematically tractable. ### 10.3 Return (Cumulative Discounted Reward) ``` G_t = r_(t+1) + γ·r_(t+2) + γ²·r_(t+3) + ... = Σ(k=0 to ∞) γ^k · r_(t+k+1) ``` * `γ = 0` → agent is fully "myopic" (only cares about immediate reward). * `γ → 1` → agent values future rewards almost as much as immediate ones (far-sighted). --- ## 11. Bellman Equations ### 11.1 Purpose The **Bellman equations** express the value of a state (or state-action pair) **recursively** in terms of the immediate reward plus the (discounted) value of the successor state — this recursive structure is the mathematical backbone of nearly every RL algorithm. ### 11.2 Bellman Expectation Equation for V(s) ``` V^π(s) = Σ_a π(a|s) · Σ_s' P(s'|s,a) · [ R(s,a,s') + γ·V^π(s') ] ``` In words: *the value of a state equals the expected immediate reward plus the discounted expected value of wherever you land next, averaged over the policy's action choices and the environment's transition randomness.* ### 11.3 Bellman Expectation Equation for Q(s,a) ``` Q^π(s,a) = Σ_s' P(s'|s,a) · [ R(s,a,s') + γ · Σ_a' π(a'|s') · Q^π(s',a') ] ``` ### 11.4 Bellman OPTIMALITY Equations Rather than averaging over a fixed policy's action probabilities, the **optimal** value functions take the **max** over actions — this is what characterizes the best possible policy: ``` V*(s) = max_a Σ_s' P(s'|s,a) · [ R(s,a,s') + γ·V*(s') ] Q*(s,a) = Σ_s' P(s'|s,a) · [ R(s,a,s') + γ · max_a' Q*(s',a') ] ``` Once `Q*(s,a)` is known, the **optimal policy** is simply: `π*(s) = argmax_a Q*(s,a)`. --- ## 12. Value Iteration and Policy Iteration ### 12.1 Value Iteration Directly and iteratively applies the **Bellman optimality equation** as an update rule until `V(s)` converges, then extracts the optimal policy at the end. ``` Initialize V(s) = 0 for all s repeat for each state s: V(s) ← max_a Σ_s' P(s'|s,a) · [ R(s,a,s') + γ·V(s') ] until V converges (change below threshold) Extract policy: π(s) = argmax_a Σ_s' P(s'|s,a)·[R(s,a,s') + γ·V(s')] ``` ### 12.2 Policy Iteration Alternates between two explicit phases: **Policy Evaluation** (compute `V^π` for the *current* fixed policy) and **Policy Improvement** (greedily update the policy using that value function) — repeated until the policy stops changing. ``` Initialize an arbitrary policy π repeat -- Policy Evaluation: solve/iterate V^π(s) = Σ_a π(a|s) Σ_s' P(s'|s,a)[R(s,a,s') + γV^π(s')] until convergence -- Policy Improvement: for each state s: π'(s) ← argmax_a Σ_s' P(s'|s,a)[R(s,a,s') + γV^π(s')] if π' == π: stop (optimal policy found) else: π ← π' until convergence ``` ### 12.3 Value Iteration vs. Policy Iteration | Parameter | Value Iteration | Policy Iteration | | :--- | :--- | :--- | | **Structure** | One combined update (implicitly improves policy every sweep). | Two explicit alternating phases (evaluate, then improve). | | **Convergence per Iteration** | Each sweep is cheap but many sweeps often needed. | Each policy-evaluation phase is more expensive, but fewer outer iterations are typically needed to reach the optimal policy. | | **Guarantee** | Converges to `V*`, from which `π*` is derived. | Converges to `π*` directly and monotonically improves at each step. | | **Common Use** | Simpler to implement; preferred for large/simple state spaces. | Preferred when policy evaluation can be solved efficiently (e.g., via linear equations). | --- ## 13. Q-Learning ### 13.1 Core Idea **Q-learning** is a **model-free**, **off-policy** algorithm — it learns the optimal action-value function `Q*(s,a)` directly from experience (sampled transitions), *without* needing to know the environment's transition probabilities `P(s'|s,a)` in advance. ### 13.2 Update Rule ``` Q(s,a) ← Q(s,a) + α · [ r + γ·max_a' Q(s',a') − Q(s,a) ] └──────────────┬──────────────┘ TD (Temporal-Difference) target α = learning rate ``` The term `r + γ·max_a' Q(s',a')` is the algorithm's current best estimate of the "true" return; the difference between this target and the current `Q(s,a)` estimate is the **TD error**, scaled by learning rate `α` and used to nudge the estimate. ### 13.3 Why "Off-Policy"? Q-learning updates `Q(s,a)` using `max_a' Q(s',a')` — the value of the **best possible** next action — **regardless of which action the agent actually takes next** (which may be exploratory/random due to ε-greedy). This means it learns about the optimal (greedy) policy while potentially *behaving* according to a different, more exploratory policy — hence "off-policy." ### 13.4 Q-Learning Algorithm ``` Initialize Q(s,a) arbitrarily (e.g., all zeros) for all s,a for each episode: initialize starting state s repeat for each step of the episode: choose action a from s using policy derived from Q (e.g., ε-greedy) take action a, observe reward r and next state s' Q(s,a) ← Q(s,a) + α[ r + γ·max_a' Q(s',a') − Q(s,a) ] s ← s' until s is terminal ``` --- ## 14. SARSA ### 14.1 Core Idea **SARSA** (State-Action-Reward-State-Action) is also a model-free, TD-based algorithm — but it is **on-policy**: it updates `Q(s,a)` using the value of the action **actually chosen next** by the current behavior policy, rather than the hypothetical best action. ### 14.2 Update Rule ``` Q(s,a) ← Q(s,a) + α · [ r + γ·Q(s',a') − Q(s,a) ] ↑ a' is the action ACTUALLY taken next (chosen by the same ε-greedy policy being followed) ``` ### 14.3 SARSA Algorithm ``` Initialize Q(s,a) arbitrarily for all s,a for each episode: initialize starting state s choose action a from s using policy derived from Q (e.g., ε-greedy) repeat for each step of the episode: take action a, observe reward r and next state s' choose action a' from s' using policy derived from Q (e.g., ε-greedy) Q(s,a) ← Q(s,a) + α[ r + γ·Q(s',a') − Q(s,a) ] s ← s' ; a ← a' until s is terminal ``` ### 14.4 Q-Learning vs. SARSA — Comparison Table | Parameter | Q-Learning | SARSA | | :--- | :--- | :--- | | **Policy Type** | Off-policy (learns optimal `Q*` regardless of behavior policy) | On-policy (learns `Q^π` for the exact policy being followed, exploration included) | | **Update Target** | `r + γ·max_a' Q(s',a')` (best possible next action) | `r + γ·Q(s',a')` (actually-chosen next action) | | **Risk-Taking Behavior** | Tends to learn the "riskier" optimal path, since it ignores exploration noise during the update itself. | Tends to learn a "safer" path, since exploration mistakes are baked into the value estimate. | | **Classic Illustration** | Learns to walk right next to a cliff (optimal but risky) in the "Cliff Walking" gridworld. | Learns a path further from the cliff (safer, accounting for occasional random exploratory steps). | --- ## 15. Actor-Critic Methods ### 15.1 Motivation **Value-based** methods (Q-learning, SARSA) learn a value function and derive a policy indirectly (e.g., greedy w.r.t. Q); **policy-based** methods directly parameterize and optimize a policy `π_θ(a|s)` using gradient ascent on expected reward, but suffer from **high variance** in their gradient estimates. **Actor-Critic** combines both ideas to get the best of each. ### 15.2 The Two Components | Component | Role | | :--- | :--- | | **Actor** | The **policy** `π_θ(a\|s)` — decides *which action to take*; updated via policy-gradient ascent. | | **Critic** | A **value function** `V_w(s)` (or `Q_w(s,a)`) — evaluates *how good* the actor's chosen actions actually were; updated via TD-learning, and used to reduce variance in the actor's gradient estimate. |
### 15.3 The Advantage Function & TD Error as Critique Instead of using the raw return (high variance) to update the actor, actor-critic uses the **TD error** `δ` as a low-variance estimate of the **advantage** — how much better an action was than the state's average expected value: ``` δ = r + γ·V_w(s') − V_w(s) -- critic's TD error ("advantage" estimate) Critic update: w ← w + α_w · δ · ∇_w V_w(s) Actor update: θ ← θ + α_θ · δ · ∇_θ log π_θ(a|s) ``` ### 15.4 Why Actor-Critic Helps * The **actor** benefits from the critic's lower-variance advantage estimate instead of a noisy full-episode return, leading to more stable, faster-converging policy-gradient updates. * The **critic** benefits from being guided by the actor's actual action choices, focusing its value estimates on the states/actions that matter under the current policy. --- ## 16. Common Interview Questions #### Q1. Why can't we just use a very large fixed-length context vector to fix the encoder-decoder bottleneck instead of attention? * **Answer**: Even a very large fixed vector still forces the model to compress a variable, potentially very long amount of information into one static representation used identically for every output word; attention instead re-computes a fresh, dynamically-weighted combination of *all* source hidden states for every output step, so it scales far better and lets the decoder focus on exactly the relevant source words at each moment, rather than relying on everything being pre-squeezed losslessly into one vector. #### Q2. Why is Q-learning considered off-policy while SARSA is on-policy? * **Answer**: Q-learning's update always bootstraps toward `max_a' Q(s',a')` — the value of the greedy/optimal action — irrespective of what action the agent's exploratory behavior policy actually takes next; it is therefore learning about a *different* (optimal) policy than the one generating its data. SARSA instead bootstraps using `Q(s',a')` for the action `a'` its own current behavior policy actually selects next, so it is learning about (evaluating and improving) the very same policy it uses to act. #### Q3. What is the intuition behind the Bellman equation? * **Answer**: It captures the idea that the value of being in a state can be broken down into "one step of immediate reward now" plus "the (discounted) value of wherever you end up next" — this recursive decomposition means you never have to explicitly sum an infinite reward sequence; you only ever need to relate a state's value to its immediate neighbors' values, which is what makes dynamic-programming-style iterative solutions (Value/Policy Iteration) possible. #### Q4. Why does Actor-Critic tend to be more stable than plain policy-gradient (REINFORCE)? * **Answer**: Plain policy-gradient methods use the full, raw episodic return as their learning signal, which is an extremely high-variance, noisy quantity (it depends on the entire stochastic trajectory of rewards). Actor-Critic instead uses the critic's TD-error as a much lower-variance proxy for how much better-than-average an action was, which substantially reduces the variance of the gradient estimate used to update the actor, leading to smoother, more stable, and typically faster convergence. --- ## 18. One-Page Quick Revision Sheet ### RNN / LSTM / GRU Cheat Sheet * **RNN**: shares weights across time steps; suffers vanishing/exploding gradients on long sequences. * **LSTM**: 3 gates (forget, update/input, output) + separate cell state; additive cell update fixes vanishing gradients. * **GRU**: 2 gates (reset, update); merges cell & hidden state; fewer parameters, faster to train. ### Translation / Attention Cheat Sheet * **Encoder-Decoder**: encoder compresses source → context vector → decoder generates target. * **Beam Search**: keeps top-B partial sequences at each step; B=1 → greedy; larger B → better but costlier; use length normalization. * **BLEU**: clipped n-gram precision × brevity penalty; measures surface overlap with reference translation(s), not meaning. * **Attention**: computes a fresh weighted sum over *all* encoder states per decoder step — removes the fixed-context-vector bottleneck. ### Reinforcement Learning Cheat Sheet * **MDP**: (S, A, P, R, γ) — memoryless (Markov) transitions. * **Bellman Equation**: value = immediate reward + discounted value of next state (recursive). * **Value Iteration**: iterate Bellman *optimality* update directly on V(s). * **Policy Iteration**: alternate full policy evaluation ↔ greedy policy improvement. * **Q-learning**: off-policy, TD target uses `max_a' Q(s',a')`. * **SARSA**: on-policy, TD target uses actually-chosen `Q(s',a')`. * **Actor-Critic**: actor = policy (π_θ), critic = value function; critic's TD-error reduces variance of actor's gradient update.

# Machine Learning — Unit 5 Study Notes ## Support Vector Machines, Bayesian Learning & Applications of Machine Learning --- ## 1. Support Vector Machines (SVM) ### 1.1 Introduction **Support Vector Machines** are a supervised learning algorithm, primarily used for **classification** (and adaptable for regression as SVR), that finds the **optimal separating hyperplane** between classes — "optimal" specifically meaning the hyperplane that maximizes the **margin** (distance) to the nearest data points of each class. ### 1.2 The Hyperplane For a `d`-dimensional feature space, a hyperplane is a `(d-1)`-dimensional flat decision boundary defined by: ``` w · x + b = 0 where: w = weight vector (normal to the hyperplane) b = bias/intercept term x = input feature vector ``` Classification decision: `ŷ = sign(w · x + b)`. ### 1.3 Margin & Support Vectors The **margin** is the perpendicular distance from the separating hyperplane to the closest data point(s) of either class. The specific data points that lie exactly on the margin boundary — the ones that "support" (determine) the position of the hyperplane — are called **support vectors**. Only these points affect the final decision boundary; all other points could be moved or removed without changing the solution.
### 1.4 Maximum Margin Optimization Problem The margin width equals `2/||w||`; maximizing the margin is therefore equivalent to **minimizing `||w||`** (or `½||w||²` for a convenient convex quadratic form), subject to every point being correctly classified with margin ≥ 1: ``` minimize: ½ ||w||² subject to: y_i (w · x_i + b) ≥ 1 for every training point (x_i, y_i), y_i ∈ {-1, +1} ``` This is a **convex quadratic programming** problem, guaranteeing a unique global optimum (no local-minima issues, unlike many neural network objectives). ### 1.5 Hard Margin vs. Soft Margin SVM | Type | Assumption | Handles Non-Separable Data? | | :--- | :--- | :--- | | **Hard Margin SVM** | Data is perfectly linearly separable; **no** misclassification allowed. | No — fails/undefined if classes overlap even slightly. | | **Soft Margin SVM** | Allows some points to violate the margin (or even be misclassified), controlled by **slack variables** `ξ_i ≥ 0`. | Yes — practical for real-world noisy/overlapping data. | ### 1.6 Soft Margin Formulation ``` minimize: ½ ||w||² + C · Σ ξ_i subject to: y_i (w · x_i + b) ≥ 1 - ξ_i, ξ_i ≥ 0 for all i ``` * **`ξ_i` (slack variable)**: how far point `i` is allowed to violate the margin (0 = no violation). * **`C` (regularization / penalty parameter)**: controls the trade-off between maximizing margin width and minimizing margin violations. | C value | Effect | Risk | | :--- | :--- | :--- | | **Large C** | Heavily penalizes misclassification → narrower margin, fits training data closely. | Higher risk of **overfitting**. | | **Small C** | Tolerates more margin violations → wider margin, smoother boundary. | Higher risk of **underfitting**. | ### 1.7 The Kernel Trick Many real datasets are **not linearly separable** in their original feature space. The **kernel trick** implicitly maps input features into a much higher-dimensional space where a linear separator *does* exist — without ever explicitly computing the (potentially infinite-dimensional) transformed coordinates, by replacing every dot-product `x_i · x_j` in the SVM optimization with a **kernel function** `K(x_i, x_j)`.
### 1.8 Common Kernel Functions | Kernel | Formula | Use Case | | :--- | :--- | :--- | | **Linear** | `K(x_i,x_j) = x_i · x_j` | Data already (near) linearly separable; fast, few parameters. | | **Polynomial** | `K(x_i,x_j) = (x_i · x_j + c)^d` | Captures interaction/curvature up to degree `d`. | | **Radial Basis Function (RBF / Gaussian)** | `K(x_i,x_j) = exp(-γ‖x_i - x_j‖²)` | Most popular general-purpose kernel; maps to infinite-dimensional space; good default choice. | | **Sigmoid** | `K(x_i,x_j) = tanh(κ·x_i·x_j + c)` | Related to neural-network activation; less common in practice. | ### 1.9 Why the Kernel Trick is Efficient The optimization problem's **dual formulation** depends on training data *only* through pairwise dot products `x_i · x_j` — never through the individual feature vectors themselves — so substituting `K(x_i,x_j)` for that dot product transparently "upgrades" the whole algorithm to operate in the higher-dimensional space, at the *same* computational cost as the original low-dimensional dot product, avoiding the "curse of dimensionality" explicit computation would otherwise incur. ### 1.10 SVM for Multi-Class Classification SVMs are inherently **binary** classifiers; multi-class problems are handled via decomposition strategies: | Strategy | Approach | | :--- | :--- | | **One-vs-Rest (OvR)** | Train `k` binary classifiers, each separating one class from all others combined; pick the class whose classifier gives the highest confidence score. | | **One-vs-One (OvO)** | Train `k(k-1)/2` binary classifiers, one per class pair; classify by majority vote among all pairwise classifiers. | ### 1.11 Advantages & Disadvantages of SVM | Advantages | Disadvantages | | :--- | :--- | | Effective in high-dimensional spaces, even when dimensions > samples. | Training time scales poorly (often O(n²)–O(n³)) on very large datasets. | | Memory-efficient at inference (only support vectors matter). | Choice of kernel and hyperparameters (C, γ) requires careful tuning. | | Convex optimization → guaranteed global optimum, no local minima. | Does not directly provide probabilistic class-membership estimates (needs extra calibration, e.g., Platt scaling). | | Works well with a clear margin of separation. | Less interpretable than simpler models like decision trees. | --- ## 2. Bayesian Learning ### 2.1 Introduction **Bayesian learning** treats model parameters (and hypotheses) as **random variables** with associated probability distributions, and uses **Bayes' Theorem** to update beliefs about these parameters as evidence (data) is observed — providing a principled framework to incorporate prior knowledge and quantify uncertainty in predictions. ### 2.2 Bayes' Theorem ``` P(D | h) · P(h) P(h | D) = ───────────────────── P(D) P(h|D) = Posterior probability — probability of hypothesis h, given observed data D P(D|h) = Likelihood — probability of observing D, if h were true P(h) = Prior probability — initial belief in h, before seeing any data P(D) = Evidence (marginal) — total probability of D, over all hypotheses: Σ_h P(D|h)P(h) ```
### 2.3 MAP (Maximum A Posteriori) Hypothesis Rather than computing the full posterior distribution, we often just want the **single most probable hypothesis** given the data: ``` h_MAP = argmax_h P(h | D) = argmax_h P(D | h) · P(h) ``` (The denominator `P(D)` is dropped since it's constant across all hypotheses and doesn't affect the argmax.) ### 2.4 MLE (Maximum Likelihood Estimate) If we assume a **uniform prior** (all hypotheses equally likely a priori), MAP reduces exactly to choosing the hypothesis that best explains the data alone: ``` h_ML = argmax_h P(D | h) ``` ### 2.5 MAP vs. MLE — Comparison Table | Parameter | MLE | MAP | | :--- | :--- | :--- | | **Uses Prior?** | No (implicitly assumes uniform/uninformative prior). | Yes — explicitly incorporates prior belief `P(h)`. | | **Formula** | `argmax_h P(D\|h)` | `argmax_h P(D\|h)·P(h)` | | **Behavior with Little Data** | Can overfit badly / be unstable with small sample sizes. | Prior regularizes the estimate, more stable with limited data. | | **Behavior with Lots of Data** | Converges to the true parameter as data → ∞. | Also converges to MLE as data → ∞ (prior's influence washes out). | ### 2.6 Naïve Bayes Classifier A practical, highly efficient classifier built directly from Bayes' theorem, applying the **"naïve" conditional independence assumption**: all features are assumed **conditionally independent of each other, given the class label**. ``` P(class | x_1,...,x_n) ∝ P(class) · Π(i=1 to n) P(x_i | class) Prediction: ŷ = argmax_class P(class) · Π_i P(x_i | class) ``` ### 2.7 Why "Naïve"? The assumption that features are conditionally independent given the class is almost never literally true in real data (features are usually correlated), yet Naïve Bayes performs surprisingly well in practice, especially for **text classification** — because what matters for classification accuracy is often just getting the *ranking/argmax* of class probabilities right, not the exact probability values. ### 2.8 Types of Naïve Bayes (by feature/likelihood distribution) | Variant | Assumed Feature Distribution | Typical Use Case | | :--- | :--- | :--- | | **Gaussian Naïve Bayes** | Continuous features, assumed normally distributed per class. | General continuous-valued data. | | **Multinomial Naïve Bayes** | Discrete counts (e.g., word frequency counts). | Text classification, spam filtering. | | **Bernoulli Naïve Bayes** | Binary/boolean features (presence/absence). | Text classification using word presence rather than frequency. | ### 2.9 Bayesian (Belief) Networks A **Bayesian Network** is a **directed acyclic graph (DAG)** where nodes represent random variables and directed edges represent direct probabilistic (causal/conditional) dependencies — it compactly encodes the **joint probability distribution** over all variables as a product of local conditional probabilities. ``` P(X_1, X_2, ..., X_n) = Π(i=1 to n) P( X_i | Parents(X_i) ) ```
### 2.10 Reading the Network In the example above: `P(Rain, Sprinkler, Cloudy, WetGrass) = P(Cloudy)·P(Rain|Cloudy)·P(Sprinkler|Cloudy)·P(WetGrass|Rain,Sprinkler)`. Each node's probability depends **only** on its direct parents — this is the **local Markov property** of Bayesian networks, and it is what makes the joint distribution tractable to represent instead of needing an exponentially large full joint probability table. ### 2.11 Naïve Bayes as a Special-Case Bayesian Network Naïve Bayes can be viewed as a very simple Bayesian network: a single "class" node as the parent, with every feature node as its child, and **no edges between feature nodes** — visually encoding the conditional-independence-given-class assumption directly in the graph structure. --- ## 3. Why does SVM use only the support vectors to define the decision boundary, ignoring other points? * **Answer**: The margin-maximization objective is defined purely in terms of the closest points to the boundary from each class; all other points lie further from the boundary than these support vectors and satisfy their margin constraint with room to spare, so they exert no binding force on where the optimal hyperplane sits — mathematically, in the dual optimization formulation, non-support-vector points end up with a Lagrange multiplier of exactly zero, meaning they simply drop out of the solution entirely. #### Q2. What's the difference between a generative and discriminative approach, and where does Naïve Bayes fit? * **Answer**: A discriminative model learns the decision boundary or `P(class|features)` directly, whereas a generative model like Naïve Bayes learns `P(features|class)` and `P(class)` separately, then uses Bayes' theorem to derive the posterior — this means a generative model like Naïve Bayes can, in principle, also be used to generate/sample new synthetic data resembling each class, which a purely discriminative model like a standard SVM cannot do. #### Q3. Why did deep learning suddenly succeed at ImageNet-scale in 2012 rather than decades earlier, when neural networks had already existed? * **Answer**: The core ideas behind CNNs existed well before 2012, but three factors converged specifically around that time: sufficiently large labeled datasets (ImageNet's millions of images) to avoid severe overfitting in deep networks, sufficient computational power (GPUs) to make training such deep networks computationally practical within reasonable time, and algorithmic refinements (ReLU activations, dropout) that made very deep networks actually trainable — without any one of these three factors, the others alone would likely not have produced AlexNet's breakthrough result. #### Q4. Why is the kernel trick considered computationally efficient despite mapping to a much higher-dimensional (sometimes infinite-dimensional) space? * **Answer**: The SVM's dual optimization problem never needs the explicit coordinates of data points in the higher-dimensional space — it only ever needs the pairwise dot-product/similarity between points, and the kernel function computes that similarity directly in the original lower-dimensional input space without ever materializing the transformed vectors, so the computational cost stays proportional to the original feature dimensionality rather than the (possibly infinite) transformed dimensionality. --- ## 7. One-Page Quick Revision Sheet ### SVM Cheat Sheet * **Hyperplane**: `w·x + b = 0`; margin width = `2/||w||`. * **Support Vectors**: the closest points to the boundary; only these determine the solution. * **Hard Margin**: no violations allowed (needs perfectly separable data). * **Soft Margin**: slack variables `ξ_i` + penalty `C` allow controlled violations. * **Kernel Trick**: replace dot product with `K(x_i,x_j)` to implicitly work in higher-dimensional space (Linear, Polynomial, RBF, Sigmoid). ### Bayesian Learning Cheat Sheet * **Bayes' Theorem**: `P(h|D) = P(D|h)·P(h) / P(D)`. * **MAP**: `argmax_h P(D|h)·P(h)` — uses prior. * **MLE**: `argmax_h P(D|h)` — assumes uniform prior. * **Naïve Bayes**: assumes features conditionally independent given class; variants — Gaussian, Multinomial, Bernoulli. * **Bayesian Network**: DAG encoding joint distribution as product of local conditionals `P(X_i | Parents(X_i))`. ### ML Applications Cheat Sheet * **Computer Vision**: classification, detection, segmentation, face recognition — powered by CNNs. * **Speech Processing**: ASR, speaker recognition, TTS — typically operates on MFCC/spectrogram features. * **NLP**: classification, NER, translation, QA, generation — now dominated by Transformer architectures. * **Common Pattern**: large-scale pretraining followed by task-specific fine-tuning (transfer learning) across all three domains. ### ImageNet Case Study Cheat Sheet * **ImageNet**: large labeled image database (WordNet hierarchy); **ILSVRC** = annual 1000-class competition benchmark starting 2010. * **Metric**: Top-5 error rate. * **2012 — AlexNet**: first deep CNN winner, ~10-point error drop; started the deep learning revolution (ReLU, dropout, GPUs, augmentation). * **Evolution**: AlexNet (2012) → ZFNet (2013) → VGGNet / GoogLeNet (2014) → ResNet (2015, residual connections, surpassed human-level accuracy). * **Legacy**: ImageNet-pretrained models became the standard transfer-learning backbone across nearly all of computer vision.
